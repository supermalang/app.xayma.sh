Animation Philosophy

All animations follow design.md motion rules:

  • ✓ Only for state transitions (clarifying, never distracting)
  • ✓ Easing: expo-out (fast → slow), no bounce
  • ✓ No layout property animations (opacity + transform only)
  • ✓ Timing: 150–300ms standard, 350ms for large shifts
  • ✓ Stagger creates scanning effect (guides visual flow)
  • ✓ Hover + focus states smooth (250ms transitions)

Deployments Page Animations

Operational page focused on deployment status clarity and action responsiveness.

1. Page Header (Permanent)

Title + "Create Deployment" button appear immediately (300ms fade).

2. Admin Stats Cards (Staggered)

Total Deployments → Active Deployments → Monthly Cost, each 80ms apart (same as Dashboard).

Timing: 300ms fade + 80ms stagger
Easing: expo-out
Trigger: Page load (if user is admin)
Effect: Opacity 0→1 + TranslateY 8px→0

3. Status Filter Bar (Slide-Up)

Filter appears after stats, slight upward motion.

Timing: 300ms (100ms delay from header)
Easing: expo-out
Effect: Opacity 0→1 + TranslateY -6px→0
On filter change: Smooth 250ms fade transition

4. DataTable (Fade-In)

Entire table appears after filter bar is ready.

Timing: 300ms (200ms delay from header)
Easing: expo-out
Effect: Opacity 0→1 (no transform—table is complex)
On data change: Smooth re-flow with fade

5. Status Badge Pulse (Critical States)

Badges for "deploying" and "failed" status gentle pulse (draws attention, not alarming).

Timing: 1.5s infinite cycle
Easing: ease-in-out
Effect: Opacity 1.0 ↔ 0.7 (50% amplitude)
Usage: Draws eyes to critical states without distraction

6. Action Button Hover

View and Delete buttons scale slightly on hover (responsive tactile feedback).

Timing: 250ms
Easing: expo-out
Effect: Transform scale(1.0) → scale(1.1)
Platform feedback: Confirms button is interactive

Deployment Page Load Timeline

0ms → Header enters (title + create button) 0–100ms → Stats cards stagger in (if admin) 100ms → Filter bar enters 200ms → Table fades in with data ∞ → Status badges pulse (for critical states)

Total initial sequence: ~500ms to full page visibility. User can interact immediately.

Partners Page Animations

Management page focused on partner relationship oversight with clear status indicators.

1. Page Header (Permanent)

Title + "Create Partner" button appear immediately (300ms fade).

2. Filter Dropdowns (Slide-Up)

Status and Type filter dropdowns appear after header with 100ms delay.

Timing: 300ms (100ms delay from header)
Easing: expo-out
Effect: Opacity 0→1 + TranslateY -6px→0
On change: Smooth 250ms fade transition

3. Partner Table (Fade-In)

DataTable appears after filters settle (200ms delay).

Timing: 300ms (200ms delay from header)
Easing: expo-out
Effect: Opacity 0→1 (no transform—table is complex)
Columns: Name, Email, Phone, Type, Status, Remaining Credits

4. Status Badge Pulse (Suspended)

Suspended status badges (warning state) gently pulse—draws attention to accounts needing action.

Timing: 1.5s infinite cycle
Easing: ease-in-out
Effect: Opacity 1.0 ↔ 0.7 (50% amplitude)
Usage: Active, Inactive have no pulse—only suspended pulses

5. Create/Edit Dialog (Slide-Up Modal)

Dialog enters from bottom when user clicks "Create" or edit pencil icon.

Timing: 300ms enter, 250ms leave
Easing: expo-out
Effect: Opacity 0→1 + TranslateY 20px→0
Form inside: PartnerForm component with validation feedback

6. Action Buttons Hover

View, Edit, Delete buttons scale slightly and smooth on hover (250ms).

Timing: 250ms
Easing: expo-out
Effect: Transform scale(1.0) → scale(1.1)
Interactive feedback: Confirms button responsiveness
